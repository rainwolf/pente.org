Hours of searching lead to frustration, frustration leads to overdue creation, creation leads to the dark side of the spam. die spam, die (it's german ;-) )

BlueFrog is dead. Okopipi doesn't produce mucho. But knujon actually wants all of our junkmail and tells us the spammers' days are counted. My GMail-account catches a lot of spam, yet there was no easy way to report tons of spam in the spam folder of my gmail account. But thanks to Python and a cute little library libgmail, it was possible to create a script to do just that.

Gspamcop is basically the same script as gknujon, except that it only reports 48hrs old spam, instead of doing a lot of filtering and putting it all in one script I decided to split them up. Since I figured there must be a lot of knujon users who don't use spamcop and/or vice versa. The idea, at least for people who use spamcop and knujon, is to run this script first, so that all the fresh spam gets picked out, which can be reported to knujon and spamcop in one go, then run the gknujon script for the rest.

To run the scripts you'll need a thing or two or three. You 'll need to install

  • Python
in that order. The StandAlone version does not need python. Make sure IMAP is enabled on your gmail-account.
